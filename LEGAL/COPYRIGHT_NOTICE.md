# Fox ML Infrastructure — Copyright Notice Requirements

**All files in the FoxML Core repository must include copyright notices.**

---

## 1. Copyright Notice Format

### 1.1 Standard Copyright Header

**All source code files should include:**

```python
# Copyright (c) 2025-2026 Fox ML Infrastructure LLC
# All rights reserved.
```

**Or for multi-line format:**

```python
"""
Copyright (c) 2025-2026 Fox ML Infrastructure LLC
All rights reserved.

[File description]
"""
```

### 1.2 License Reference

**Files should also reference the license:**

```python
# Copyright (c) 2025-2026 Fox ML Infrastructure LLC
# All rights reserved.
# Licensed under source-available license (see LICENSE) or Commercial License (see COMMERCIAL_LICENSE.md)
```

---

## 2. File Types Requiring Copyright Notices

### 2.1 Required Files

**Copyright notices are required for:**

- ✅ Python source files (`.py`)
- ✅ Configuration files (`.yaml`, `.yml`, `.json`)
- ✅ Documentation files (`.md`)
- ✅ Shell scripts (`.sh`)
- ✅ C++ source files (`.cpp`, `.h`, `.hpp`)
- ✅ Any other source code files

### 2.2 Optional Files

**Copyright notices are recommended for:**

- Build scripts
- Test files
- Example files
- Documentation templates

---

## 3. Copyright Notice Placement

### 3.1 Source Code Files

**Place copyright notices at the top of the file:**

```python
# Copyright (c) 2025-2026 Fox ML Infrastructure LLC
# All rights reserved.

import os
import sys
# ... rest of code
```

### 3.2 Documentation Files

**Place copyright notices at the bottom of the file:**

```markdown
# Document Title

[Document content]

---

**Copyright (c) 2025-2026 Fox ML Infrastructure LLC**  
**All rights reserved.**
```

---

## 4. Removing or Altering Copyright Notices

### 4.1 Prohibited

**You MAY NOT:**

- ❌ Remove copyright notices from any files
- ❌ Alter copyright notices
- ❌ Replace copyright notices with your own
- ❌ Claim authorship of the software

**Removing or altering copyright notices is a license violation.**

**See `COMMERCIAL_LICENSE.md` Section 4 (Restrictions) and `LEGAL/LICENSE_ENFORCEMENT.md` Section 7.**
