# Internal Implementation Details

This folder contains internal implementation details, fix documentation, and analysis documents that are primarily for developers working on the codebase.

## Contents

- **Fix Documentation**: Detailed fixes for specific bugs or issues
- **Analysis Documents**: Deep-dive analysis of system behavior
- **Review Documents**: Code reviews and implementation reviews
- **Progress Reports**: Implementation progress tracking
- **Design Comparisons**: Design decision comparisons

These documents are **internal** and may contain:
- Implementation details that change frequently
- Debugging information
- Historical context for fixes
- Design decision rationale

For user-facing documentation, see the parent `training_utils/` directory.
